برای ساخت یک برنامه با استفاده از `create-t3-app`، هر یک از دستورات زیر را اجرا کنید و به سوالات خط فرمان پاسخ دهید:

### npm

```bash
npm create t3-app@latest
```

### yarn

```bash
yarn create t3-app
```

### pnpm

```bash
pnpm create t3-app@latest
```

### bun

```bash
bun create t3-app@latest
```

پس از ساخت برنامه شما، [اولین مراحل](/fa/usage/first-steps) را بررسی کنید تا برنامه جدید خود را شروع کنید.

## استفاده پیشرفته

| گزینه/پرچم        | توضیحات                                                                           |
| ----------------- | --------------------------------------------------------------------------------- |
| `[dir]`           | شامل یک آرگومان دایرکتوری با نام پروژه                                            |
| `--noGit`         | به صراحت به CLI بگویید که یک مخزن git جدید در پروژه راه‌اندازی نکند               |
| `-y`, `--default` | از CLI عبور کنید و یک t3-app جدید را با تمام گزینه‌های انتخاب شده راه‌اندازی کنید |
| `--noInstall`     | پروژه را بدون نصب وابستگی‌ها ایجاد کنید                                           |

## استفاده آزمایشی

برای CI ما، پرچم‌های آزمایشی داریم که به شما امکان می‌دهد هر برنامه‌ای را بدون هیچ سوالی ایجاد کنید. اگر این مورد برای شما اعمال می‌شود، می‌توانید از این پرچم‌ها استفاده کنید. لطفاً توجه داشته باشید که این پرچم‌ها آزمایشی هستند و ممکن است در آینده بدون پیروی از نسخه‌بندی semver تغییر کنند.

| پرچم                      | توضیحات                                           |
| ------------------------- | ------------------------------------------------- |
| `--CI`                    | به CLI اطلاع دهید که در حالت CI هستید             |
| `--trpc`                  | tRPC را در پروژه قرار دهید                        |
| `--prisma`                | Prisma را در پروژه قرار دهید                      |
| `--drizzle`               | Drizzle را در پروژه قرار دهید                     |
| `--nextAuth`              | NextAuth.js را در پروژه قرار دهید                 |
| `--tailwind`              | Tailwind CSS را در پروژه قرار دهید                |
| `--dbProvider [provider]` | یک پایگاه داده پیکربندی شده را در پروژه قرار دهید |
| `--appRouter`             | از Next.js App Router در پروژه استفاده کنید       |

<Callout type="warning">
  اگر پرچم `CI` را ارائه نکنید، بقیه این پرچم‌ها هیچ تاثیری ندارند.
</Callout>

شما نیازی به صراحتاً انصراف از بسته‌هایی که نمی‌خواهید ندارید. با این حال، اگر ترجیح می‌دهید صریح باشید، می‌توانید `false` را ارسال کنید، به عنوان مثال `--nextAuth false`.

دستور `--dbProvider` دارای 4 مقدار پایگاه داده برای انتخاب است: mysql، postgres، planetscale، sqlite. اگر دستور ارائه نشود، مقدار پیش‌فرض sqlite خواهد بود.

### مثال

مثال زیر یک برنامه T3 با tRPC و Tailwind CSS ایجاد می‌کند.

```bash
pnpm dlx create-t3-app@latest --CI --trpc --tailwind
```

مثال زیر یک برنامه T3 با NextAuth.js، Tailwind CSS، Drizzle و PostgreSQL ایجاد می‌کند.

```bash
pnpm dlx create-t3-app@latest --CI --nextAuth --tailwind --drizzle --dbProvider postgres
```
